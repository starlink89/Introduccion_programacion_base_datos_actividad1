1. ¿Qué es un Dato?

Un dato es la manera en que representamos una variable, ya sea con números o con palabras, para mostrar el valor que se le da a algo. Se puede representar con una serie de símbolos, números o letras.

2. ¿Qué es información?

La información es un conjunto de datos organizados y procesados que forman un mensaje. Este mensaje puede cambiar lo que una persona o sistema sabe sobre algo.

3. ¿Diferencia entre dato e información?

Los términos de datos e información se usan para describir lo mismo, para nosotros los informáticos éstos términos significan diferentes cosas.

Datos es un término que se refiere a hechos, eventos, transacciones, etc., que han sido registrados. Es la entrada sin procesar de la cual se produce la información.

Información se refiere a los datos que han sido procesados y comunicados de tal manera que pueden ser entendidos e interpretados por el receptor.

5. ¿Qué es un modelado de Datos?

El modelado de datos es el proceso de crear un esquema o representación visual que define como una organización recopila y gestiona su información. Este modelo ayuda a los analistas de datos, científicos e ingenieros a tener una visión unificada de los datos de la empresa. En el modelo se detalla qué datos recoge la empresa, cómo se relacionan entre sí y cómo se van a almacenar y analizar.

6. ¿Tipos de bases de Datos, según:

  • Variabilidad: Bases de datos estáticas y dinámicas.
  • Contenido: Base de datos bibliográficas, texto completo y multimedia.
  • Modelo de base de datos: Base de datos Jerárquicas, en red, relacionales, orientadas a objetos, documentales, grafos y NoSQL.

7. ¿Qué es el modelo relacional?

El modelo relacional, propuesto por Edgar Frank Codd en 1970, organiza y gestiona bases de datos usando lógica y teoría de conjuntos. Se basa en usar relaciones, que se pueden imaginar como tablas con filas (tuplas) y columnas (campos). Este modelo es el más utilizado hoy en día para resolver problemas reales y manejar datos de manera dinámica, creando un esquema de base de datos que se implementa en sistemas de gestión de bases de datos (DBMS).

8. ¿Describa los elementos del Diagrama entidad Relacional?

  • Entidades: Son los objetos o cosas que queremos almacenar información, representadas por rectángulos. Ejemplo: "Cliente" o "Producto".
  • Atributos: Son las características de las entidades, representados por óvalos. Ejemplo: el nombre y la dirección de un cliente.
  • Relaciones: Son las conexiones entre entidades, representadas por rombos. Ejemplo: "Compra" conecta "Cliente" y "Producto".
  • Llaves primarias: Son los identificadores únicos de cada entidad, subrayados en los atributos. Ejemplo: el ID del cliente.
  • Llaves foráneas: Son los atributos que crean una relación entre entidades, usualmente claves primarias de otra entidad.

9. ¿Qué es una entidad?

10. ¿Qué es un atributo y sus tipos?

Los atributos en bases de datos son las características que describen a una entidad. Los tipos más comunes son:

  • Simples: No se dividen más. Ejemplo: Nombre.
  • Compuestos: Se pueden dividir en partes más pequeñas. Ejemplo: Dirección (calle, ciudad).
  • Mono valorados: Tienen un solo valor. Ejemplo: Número de identificación.
  • Multivalorados: Pueden tener varios valores. Ejemplo: Números de teléfono.
  • Derivados: Se calculan a partir de otros datos. Ejemplo: Edad (a partir de la fecha de nacimiento).
  • Clave: Identifican de manera única a una entidad. Ejemplo: ID de empleado.

11. ¿Qué es una relación?

Son las conexiones entre entidades, representadas por rombos. Ejemplo: "Compra" conecta "Cliente" y "Producto".

12. ¿Qué son las llaves primarias, foránea y sus características?

Llaves primarias: Son los identificadores únicos de cada entidad, subrayados en los atributos. Ejemplo: el ID del cliente. Llaves foráneas: Son los atributos que crean una relación entre entidades, usualmente claves primarias de otra entidad.

13. ¿Qué son las reglas del Negocio?

En el contexto de bases de datos, una regla de negocio es una norma que define o limita cómo se deben manejar los datos. Su propósito es establecer las directrices para la estructura y funcionamiento de la base de datos, así como para controlar o influir en el manejo de la información.

14. ¿Qué es la cardinalidad o mapeo?

Cardinalidad es el número de entidades con la cual otra entidad puede asociar mediante una relación binaria; la cardinalidad puede ser: Uno a uno, uno a muchos o muchos a uno y muchos a muchos.
